# תקציב פרויקטים קהילתי

אפליקציית ווב פשוטה לבחירת פרויקטים מתוך רשימה, עם תקציב מרבי, שמתאימה עד 500 משתמשים. הבחירה של כל משתמש נשמרת בדפדפן שלו בלבד – אף אחד לא רואה את מה שאחר בחר.

## מה האפליקציה עושה

- בצד ימין: רשימת פרויקטים צבעוניים, כל אחד במחיר.
- בצד שמאל: "בנק" שמראה את הסכום הכולל של הפרויקטים שנבחרו.
- גוררים פרויקט מימין לשמאל – הסכום מתעדכן.
- כל עוד הסכום קטן או שווה לתקציב – הבנק ירוק. כשעוברים – אדום.
- עובד גם במחשב (עכבר) וגם בטלפון (מגע), תמיד מימין לשמאל.
- כפתור "מנהל" בראש הדף פותח חלון עריכה אחרי סיסמה (`123456789` כברירת מחדל).
  - אפשר לשנות את התקציב המרבי.
  - אפשר להוסיף / למחוק / לשנות פרויקטים וצבעים.
  - אפשר לאפס הכול לברירת המחדל.

## בדיקה מהירה במחשב שלך

פשוט לחצו פעמיים על `index.html` והוא ייפתח בדפדפן. לא צריך שום התקנה.

# מדריך העלאה ל-GitHub ו-GitHub Pages (צעד אחר צעד)

הכל דרך הדפדפן, בלי להתקין שום תוכנה. לוקח כ-10 דקות.

## שלב 1 – יצירת חשבון GitHub

1. היכנסו ל-[https://github.com/signup](https://github.com/signup).
2. הזינו כתובת מייל, בחרו סיסמה ושם משתמש (username) – שם המשתמש יופיע בכתובת האתר שלכם בהמשך, אז בחרו משהו שנוח להקליד (למשל `yoav-melamed`).
3. אשרו את המייל שמגיע מ-GitHub.
4. בחרו ב-Free plan כשתישאלו.

## שלב 2 – יצירת מאגר (Repository) חדש

1. אחרי ההתחברות, לחצו בפינה הימנית־עליונה על הסמל **+** ואז **New repository**
   (או פשוט לכו ל-[https://github.com/new](https://github.com/new)).
2. מלאו:
   - **Repository name**: `budget-app` (או כל שם אחר – באנגלית, בלי רווחים).
   - **Description**: אפשר להשאיר ריק או לכתוב "תקציב פרויקטים".
   - סמנו **Public** (חייב להיות Public כדי ש-GitHub Pages יעבוד בחינם).
   - אל תסמנו "Add a README" – כבר יש לנו אחד.
3. לחצו **Create repository**.

## שלב 3 – העלאת הקבצים

1. בדף המאגר שנוצר, לחצו על הקישור **"uploading an existing file"** (הוא באמצע הדף, בסעיף "Quick setup").
   - אם לא רואים אותו: לחצו על הכפתור **Add file** → **Upload files**.
2. גררו לחלון העלאה את **שני הקבצים**:
   - `index.html`
   - `README.md`
3. בתחתית הדף, בקטע "Commit changes":
   - בשדה הראשון כתבו: `Initial upload`
   - השאירו את האפשרות "Commit directly to the `main` branch" מסומנת.
4. לחצו **Commit changes**.

## שלב 4 – הפעלת GitHub Pages

1. בדף המאגר, לחצו למעלה על **Settings** (גלגל שיניים).
2. בתפריט בצד שמאל (באנגלית – `Pages`) לחצו על **Pages**.
3. בסעיף **Source** בחרו **Deploy from a branch**.
4. בסעיף **Branch** בחרו:
   - Branch: **main**
   - Folder: **/ (root)**
5. לחצו **Save**.
6. המתינו כ-1–2 דקות ורעננו את הדף. למעלה יופיע קישור בסגנון:

   ```
   https://<השם-משתמש-שלך>.github.io/budget-app/
   ```

   זהו – זו כתובת האפליקציה שלכם. אפשר לשתף אותה עם כל אחד.

## שלב 5 – עדכונים בעתיד

רוצים לשנות משהו בקוד? אפשר:

- **הדרך הקלה** – בדף המאגר, לחצו על `index.html`, לחצו על עט העריכה (pencil icon), ערכו, ובסוף לחצו **Commit changes**. האתר יתעדכן תוך דקה.
- **הדרך של מחיקה-והעלאה-מחדש** – בדף המאגר לחצו על הקובץ, בחרו את סמל פח האשפה, אשרו, ואז `Add file → Upload files` עם הגרסה החדשה.

## שאלות נפוצות

**המידע נשמר לכל אחד בנפרד?**
כן. כל משתמש שנכנס לאתר רואה רק את מה שהוא עצמו בחר. הנתונים נשמרים בדפדפן שלו (localStorage). אם יתחלף דפדפן או יעבור למצב גלישה פרטית – הוא יתחיל מאפס.

**איך משנים את הסיסמה של המנהל?**
פותחים את `index.html` (דרך עריכה ב-GitHub), מחפשים את השורה:
```js
const ADMIN_PASSWORD = '123456789';
```
ומשנים את הערך שבין הגרשיים. לוחצים **Commit changes** – תוך דקה האתר מתעדכן.

**אני רוצה להחזיר את רשימת הפרויקטים המקורית**
נכנסים למצב מנהל (כפתור "מנהל" → סיסמה) ולוחצים על **"איפוס לברירת מחדל"**. זה מחזיר גם את התקציב ל-200,000 ₪ וגם את 15 הפרויקטים המקוריים.

**אחד המשתמשים אומר שאצלו הכל נעלם – מה עושים?**
כנראה שהוא ניקה את היסטוריית הדפדפן או שהוא גולש במצב פרטי. אין מה לעשות – זה התנאי של "כל אחד רק לעצמו". אם רוצים שמירה מרכזית יש צורך ב-backend (אפשר להרחיב את האפליקציה בהמשך).
