# תקציב פרויקטים קהילתי

אפליקציית ווב פשוטה לבחירת פרויקטים מתוך רשימה בתוך תקציב נתון. מתאימה עד 500 משתמשים. הבחירה האישית של כל משתמש נשמרת **בדפדפן שלו בלבד** – אף אחד לא רואה את מה שאחר בחר.

## מה האפליקציה עושה

- בצד ימין: רשימת פרויקטים צבעוניים, כל אחד במחיר.
- בצד שמאל: "בנק" שמראה את הסכום הכולל של הפרויקטים שבחרת.
- גוררים פרויקט מימין לשמאל – הסכום מתעדכן.
- כל עוד הסכום ≤ התקציב – הבנק ירוק. כשעוברים – אדום.
- עובד במחשב (עכבר) וגם בטלפון (מגע), תמיד מימין לשמאל – לא מלמעלה למטה.
- כפתור "נקה בחירה" מנקה רק אותי, לא משתמש אחר.

אין מנהל, אין סיסמה, אין שרת. רשימת הפרויקטים והתקציב **מוגדרים בתוך הקובץ `index.html`**. אם תרצה לשנות אותם – עורכים את הקובץ (למטה יש הסבר מלא).

## בדיקה מהירה במחשב שלך

לחץ פעמיים על `index.html` – הוא ייפתח בדפדפן. לא צריך שום התקנה.

# חלק 1 – העלאה ל-GitHub ו-GitHub Pages (צעד אחר צעד)

הכל דרך הדפדפן, בלי להתקין שום תוכנה. לוקח כ-10 דקות.

## שלב 1 – יצירת חשבון GitHub

1. היכנסו ל-[https://github.com/signup](https://github.com/signup).
2. הזינו כתובת מייל, בחרו סיסמה ושם משתמש (username). שם המשתמש יופיע בכתובת האתר שלכם בהמשך (למשל `yoav-melamed`).
3. אשרו את המייל שמגיע מ-GitHub.
4. בחרו ב-Free plan כשתישאלו.

## שלב 2 – יצירת מאגר (Repository) חדש

1. אחרי ההתחברות, לחצו בפינה הימנית-עליונה על הסמל **+** ואז **New repository**
   (או לכו ישר ל-[https://github.com/new](https://github.com/new)).
2. מלאו:
   - **Repository name**: `budget-app` (או כל שם אחר – באנגלית, בלי רווחים).
   - **Description**: אפשר להשאיר ריק או לכתוב "תקציב פרויקטים".
   - סמנו **Public** (חייב להיות Public כדי ש-GitHub Pages יעבוד בחינם).
   - אל תסמנו "Add a README" – כבר יש לנו אחד.
3. לחצו **Create repository**.

## שלב 3 – העלאת הקבצים

1. בדף המאגר שנוצר, לחצו על הקישור **"uploading an existing file"** באמצע הדף.
   - אם לא רואים אותו: לחצו **Add file** → **Upload files**.
2. גררו לחלון העלאה את שני הקבצים:
   - `index.html`
   - `README.md`
3. בתחתית הדף, בקטע "Commit changes":
   - בשדה הראשון כתבו: `Initial upload`
   - השאירו מסומן: "Commit directly to the `main` branch".
4. לחצו **Commit changes**.

## שלב 4 – הפעלת GitHub Pages

1. בדף המאגר, לחצו למעלה על **Settings**.
2. בתפריט הצדדי (באנגלית), לחצו **Pages**.
3. תחת **Source** בחרו **Deploy from a branch**.
4. תחת **Branch**:
   - Branch: **main**
   - Folder: **/ (root)**
5. לחצו **Save**.
6. המתינו כ-1-2 דקות ורעננו את הדף. בראש הדף יופיע קישור בסגנון:

   ```
   https://<השם-משתמש-שלך>.github.io/budget-app/
   ```

   זהו – זו כתובת האפליקציה שלכם. שתפו אותה עם מי שרוצים.

# חלק 2 – עריכת רשימת הפרויקטים / התקציב

כל הנתונים מוגדרים בתוך `index.html`, בבלוק קוד ברור המתחיל בכותרת:

```
// BUDGET & PROJECTS — EDIT HERE
```

## איך עורכים דרך GitHub (בלי להתקין כלום)

1. בדף המאגר שלכם, לחצו על `index.html`.
2. בצד ימין-למעלה של תוכן הקובץ, לחצו על סמל העיפרון (**Edit this file**).
3. גללו למטה עד שתראו את השורות:

   ```js
   const BUDGET = 200000;
   ```

   ואחריהן:

   ```js
   const PROJECTS = [
     { id: 'p1',  name: 'עץ בכיכר',   price: 2000, color: '#16a34a' },
     { id: 'p2',  name: 'מחסן תרבות', price: 3000, color: '#0ea5e9' },
     ...
   ];
   ```

4. ערכו מה שצריך (ראו דוגמאות בהמשך).
5. גללו לתחתית הדף, בקטע "Commit changes":
   - כתבו הודעה קצרה כמו "עדכון תקציב" או "שינוי מחירים".
   - לחצו **Commit changes**.
6. תוך כמה עשרות שניות האתר יתעדכן. רענון דף יראה את הגרסה החדשה.

## דוגמאות לעריכה

### שינוי התקציב

```js
const BUDGET = 250000;   // במקום 200000
```

### שינוי מחיר של פרויקט קיים

מצאו את השורה, שנו רק את ערך `price`:

```js
{ id: 'p5', name: 'צביעת הכביש', price: 9000, color: '#f59e0b' },
```

### הוספת פרויקט חדש

הוסיפו שורה לרשימה. חשוב: תנו **id ייחודי חדש** (לדוגמה `p16`):

```js
{ id: 'p16', name: 'ספסלים בפארק', price: 6000, color: '#f43f5e' },
```

### מחיקת פרויקט

מוחקים את השורה המתאימה. משתמשים שבחרו אותו כבר – פשוט הוא ייעלם מהבחירה שלהם ברענון.
